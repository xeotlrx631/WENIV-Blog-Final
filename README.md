🚀 AI 연동 풀스택 블로그 프로젝트 (AI-Powered Blog)
본 프로젝트는 사용자 인증, CRUD 게시판, 그리고 AI 기능을 결합한 현대적인 블로그 서비스입니다. 6일간의 집중 개발을 통해 보안 통신부터 AI 연동, 정밀 디버깅까지의 전 과정을 구현했습니다.

📅 개발 타임라인 (Day 1 - Day 6)
Phase 1: 인프라 및 인증 구축 (Day 1~2)
환경 설정: React 기반 프론트엔드 환경 구축 및 Axios 통신 레이어 설계.

보안 인증: JWT(JSON Web Token) 기반의 로그인/회원가입 시스템 구현.

인터셉터 도입: apiClient를 통해 모든 요청 헤더에 Authorization: Bearer 토큰이 자동 포함되도록 아키텍처 설계.

Phase 2: 게시판 핵심 기능 (Day 3~4)
CRUD 구현: 게시글 생성(Create), 목록 및 상세 조회(Read), 수정(Update), 삭제(Delete) 기능 완비.

동적 라우팅: react-router-dom을 활용하여 UUID 기반의 상세 페이지 전환 시스템 구축.

Phase 3: AI 연동 및 기능 고도화 (Day 5)
AI 이어쓰기: OpenAI API 연동 엔드포인트를 호출하여 사용자의 글쓰기를 보조하는 기능을 Write.jsx에 통합.

보안 최적화: API Key 노출 방지를 위해 프론트엔드가 아닌 서버 측 프록시를 거치는 안전한 통신 구조 채택.

Phase 4: 트러블슈팅 및 시스템 안정화 (Day 6)
상세 페이지 로직 정밀 보정: URL 파라미터 오염 문제(ID 끝에 :1 등이 붙는 현상) 해결.

데이터 파싱 예외 처리: 서버 응답 구조 변경에 대응하는 유연한 데이터 바인딩 로직 적용.

🛠 주요 트러블슈팅 기록 (Key Troubleshooting)

1. HTTP 422 (Unprocessable Content) 에러 해결
   현상: 상세 페이지 진입 시 특정 ID에서만 422 에러가 발생하며 데이터 로딩 실패.

원인: 브라우저 URL 파라미터에 불필요한 세미콜론이나 숫자가 포함되어 서버의 UUID 검증 형식을 위반함.

해결: id.split(':')[0] 처리를 통해 순수 ID 값만 추출하여 서버에 요청하도록 로직 개선.

2. JSX Syntax Error 및 태그 정합성 문제
   현상: 코드 수정 중 닫는 태그(</div>) 누락 및 문자열 종결 오류로 인해 컴포넌트 렌더링 중단.

해결: 컴포넌트 전체 구조를 전수 검사하여 JSX 문법 에러를 0개로 해결하고, loading 상태를 활용하여 데이터 수신 전 렌더링 시도를 차단함으로써 안정성 확보.

3. API 모듈화 및 Export 이슈
   현상: createPost 등 특정 API 함수가 모듈에서 발견되지 않는 SyntaxError 발생.

해결: src/api/post.js 내의 export 구문을 정리하고, 일관된 호출 패턴으로 통일하여 코드의 확장성 확보.

## 📂 프로젝트 구조 (Project Structure)

```plaintext
src/
├── api/
│   ├── client.js       # Axios 인스턴스 및 인증 인터셉터
│   └── post.js         # 게시판 & AI API 호출 함수 정의
├── pages/
│   ├── Main.jsx        # 게시글 리스트 레이아웃
│   ├── Post-View.jsx   # 상세 보기 (정밀 디버깅 및 ID 정제 적용)
│   └── Write.jsx       # 글쓰기 및 AI 보조 기능 통합
└── App.js              # 전체 라우팅 및 앱 아키텍처
```

📝 최종 회고
이번 프로젝트를 통해 기능을 구현하는 능력뿐만 아니라, 실제 서비스에서 발생할 수 있는 데이터 오염과 문법적 예외 상황을 추적하고 해결하는 디버깅 역량을 키울 수 있었습니다. 특히 네트워크 레벨의 에러(422)와 프론트엔드 렌더링 에러를 동시에 해결하며 풀스택 서비스의 흐름을 깊이 있게 이해하게 되었습니다.
